# Arges

Arges は組織内のテキストコンテンツを自動分類・可視化するための自然言語処理アプリケーションです。

## アプリケーション概要

本アプリケーションは、組織内で生成される多様なテキストコンテンツ（例：成果物へのレビュー指摘、アプリケーションのバグチケット、ヘルプデスクへの依頼など）を自動的に分類・クラスタリングし、構成比や増減傾向を可視化する分析支援ツールです。自然言語処理技術を用いてテキストを内容別にグルーピングし、時間的なトレンドや部門ごとの特徴を把握できるようにします。

### このアプリケーションが提供する価値

- 定性的なテキスト情報を数値化し、統計的な比較・傾向分析を可能にする
- 主観や経験に依存しがちな判断を避け、一貫性のある客観的な評価を提供する
- 集計や傾向分析を自動化し、必要なときに即座に利用可能な評価結果を提示する

### 主な対象ユーザー

- 品質保証担当者：開発プロジェクトにおける成果物のレビュー指摘を監視し、品質管理の妥当性を把握したい
- プロダクトオーナー：開発中または運用中アプリケーションの不具合傾向を分析し、改善の優先度を決定したい
- ヘルプデスクマネージャ：依頼発生の時期やパターンを把握し、対応要員の配置計画に役立てたい

## 主な機能

本アプリケーションは以下の4つの機能を提供します。それぞれ独立実行が可能ですが、パイプラインとして所定の順番で連続実行もできます。各機能は明確な入力(Input)と出力(Output)を持ちます。

### クラスタリング (Clustering)

**目的:** テキスト集合をLLMによるベクトル化を行い、最適な複数クラスタに分割します。各クラスタの中心ベクトル（代表ベクトル）およびクラスタ識別子（Cluster ID）を算出します。

- **Input:** テキストデータのリスト（CSV、JSON、TXT形式）
- **Output:**
  - クラスタ情報（JSON形式）：クラスタIDと中心ベクトルのペア
  - クラスタ統計情報：各クラスタのサイズ、密度、分散
- **処理フロー:**
  1. AWS Bedrock経由でLLMによるテキストのベクトル化（Embedding生成）
  2. K-means++、階層クラスタリング、DBSCAN から自動選択
  3. エルボー法とシルエット分析によるクラスタ数の自動推定
- **使用モデル:** Amazon Titan Embeddings G1、Cohere Embed など

### グルーピング (Grouping)

**目的:** 新規テキストをLLMでベクトル化し、既存クラスタとの類似度計算により所属クラスタを判定します。信頼度スコアと共に出力します。

- **Input:**
  - 新規テキストデータのリスト
  - 既存のクラスタ情報（中心ベクトルとクラスタID）
- **Output:**
  - テキストIDとクラスタIDのマッピング（CSV形式）
  - 信頼度スコア（0-1の範囲）
  - コサイン類似度（クラスタ中心との類似度）
- **処理フロー:**
  1. AWS Bedrock経由で新規テキストのベクトル化
  2. 既存クラスタ中心ベクトルとのコサイン類似度計算
  3. 最高類似度クラスタへの分類と信頼度算出

### ラベリング (Labeling)

**目的:** 各クラスタの特徴をLLMが分析し、人間が理解しやすい自然言語ラベルを自動生成します。

- **Input:**
  - 各クラスタに属するテキストの代表サンプル（上位10件）
  - クラスタのメタデータ（サイズ、分散等）
- **Output:**
  - クラスタIDとラベル名の対応表（JSON形式）
  - ラベル生成の根拠となったキーワード一覧
  - 信頼度スコア（ラベルの適切性指標）
- **処理フロー:**
  1. 各クラスタの代表テキストをAWS Bedrockに送信
  2. Claude、Jurassic等のLLMがテキスト群の共通テーマを分析
  3. 簡潔で直感的なラベル名を生成
  4. 複数候補から最適なラベルを選択
- **使用モデル:** Anthropic Claude、AI21 Jurassic、Amazon Titan Text など

### パイプライン（Pipeline）

**目的:** テキストデータの集合から、全ての分析結果（クラスタ情報・グルーピング・ラベル）を一括で生成し、可視化レポートも出力します。

- **Input:** テキストデータのリスト
- **Output:**
  - 完全な分析結果一式（上記3機能の全出力）
  - インタラクティブな可視化ダッシュボード（HTML形式）
  - 分析サマリレポート（PDF形式）
  - 時系列トレンド分析（時間情報がある場合）
- **備考:** Clustering → Grouping → Labeling → Visualization の順に実行

### 可視化 (Visualization)

**目的:** 分析結果を直感的に理解できるグラフやチャートで表示します。

- **提供する可視化:**
  - クラスタ分布の2D/3D散布図
  - クラスタサイズの円グラフ
  - 時系列でのクラスタ構成変化（トレンドグラフ）
  - ワードクラウド（各クラスタの特徴語）
  - ヒートマップ（クラスタ間類似度）

## 技術仕様

### 核となる技術

- **言語**: Python 3.9+
- **クラウドプラットフォーム**: Amazon Web Services (AWS)
- **LLMサービス**:
  - **AWS Bedrock**: 統合LLMプラットフォーム
  - **対応モデル**:
    - Amazon Titan Embeddings G1（テキストベクトル化）
    - Amazon Titan Text G1（ラベル生成）
    - Anthropic Claude 3 (Sonnet/Haiku)（高精度ラベル生成）
    - Cohere Command R+（多言語対応）
    - AI21 Jurassic-2（専門用語理解）
- **エージェントフレームワーク**:
  - **AWS Strands Agents**: マルチステップ分析処理の自動化
    - **活用機能**:
      - クラスタリング→グルーピング→ラベリングの段階的実行
      - ユーザーとの対話型分析セッション
      - 分析結果の品質検証と改善提案
      - 複数データソースの統合分析
- **AWS SDK**:
  - boto3（AWS API操作）
  - AWS認証・認可機能
- **機械学習・数値計算**:
  - scikit-learn（クラスタリングアルゴリズム）
  - NumPy、SciPy（数値計算、距離計算）
- **可視化**:
  - Plotly（インタラクティブグラフ）
  - Matplotlib、Seaborn（静的グラフ）
  - WordCloud（ワードクラウド生成）
- **データ処理**:
  - pandas（データ操作）
  - 最小限のテキスト前処理（正規化、クリーニング）

### AWS利用サービス

- **Amazon Bedrock**: LLMモデルの実行基盤
- **AWS Strands Agents**: 分析ワークフローの自動実行・監視
- **Amazon S3**: 大容量データ・結果ファイルの保存
- **AWS Lambda**: バッチ処理・スケジュール実行（オプション）
- **Amazon CloudWatch**: ログ管理・監視

### パフォーマンス指標

- **処理能力**: 最大10万件のテキストを同時処理可能
- **応答時間** (AWS Bedrock利用時):
  - 1,000件以下: 2-3分以内（ベクトル化含む）
  - 10,000件以下: 15-20分以内
  - 100,000件以下: 2-3時間以内
- **メモリ使用量**: テキスト1,000件あたり約150MB（ベクトル保存含む）
- **コスト効率**:
  - Embeddings生成: 約$0.0001/1000トークン
  - ラベル生成: 約$0.003/1000トークン（Claude使用時）
- **精度指標**:
  - クラスタリング精度: シルエットスコア 0.7以上（LLM Embeddings使用）
  - ラベル適合率: 人間評価で85%以上の妥当性（LLM生成）

## 前提条件

このアプリケーションを利用するために必要な前提条件：

- **Python**: 3.9 以上
- **AWS アカウント**: Amazon Bedrock利用可能なリージョン
- **AWS 認証情報**: IAMユーザーまたはロールでBedrock権限が必要
- **メモリ**: 4GB 以上推奨（LLMベクトル処理のため）
- **ディスク容量**: 2GB 以上の空き容量（ベクトルデータ保存用）

### 必要に応じて設定が必要な項目

- **AWS認証情報**:
  - **AWS Access Key ID / Secret Access Key**: プログラマティックアクセス用
  - **IAM権限**: `bedrock:InvokeModel`, `bedrock:GetModel` 権限が必要
  - **リージョン設定**: Amazon Bedrock対応リージョン（us-east-1, us-west-2等）
- **Amazon Bedrock**:
  - **モデルアクセス申請**: 使用予定モデル（Claude, Titan等）の事前申請
  - **コスト管理**: 使用量監視・予算設定の推奨
- **入力データ要件**:
  - **対応形式**: CSV、JSON、TXT、Excel（.xlsx）
  - **文字エンコーディング**: UTF-8必須
  - **最小データ量**: 分析精度確保のため100件以上を推奨（LLM処理のため）
  - **最大ファイルサイズ**: 50MB（AWS API制限考慮）
  - **トークン制限**: 1件あたり最大2,000トークン推奨
- **多言語対応**:
  - **フルサポート**: 日本語、英語（AWS Bedrockネイティブ対応）
  - **部分サポート**: 中国語、韓国語、スペイン語、フランス語（モデル依存）

> **注意**: 開発環境のセットアップについては [開発ガイド](development.md) をご覧ください。
